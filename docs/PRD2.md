제시해주신 방향성이 매우 훌륭합니다. 단순한 '정보 제공'을 넘어, 영업사원이 현장에서 즉시 무기로 사용할 수 있는 '위기-기회 매칭 및 세일즈 자동화 도구'로 격상시키는 것이 핵심이네요.

기존에 잘 만들어두신 '행정동 검색' 및 '업종 3단 분류' 기능은 완벽하게 보존하면서, 새로 도입할 4개의 공공데이터 API를 적재적소에 배치하여 Cursor AI가 즉시 코드로 구현할 수 있도록 최고 수준의 구체성을 담은 PRD를 재작성해 드립니다.

📄 [PRD] 우체국 B2B 스마트 영업 네비게이터 고도화
부제: 공공데이터 기반 위기-기회 매칭 및 세일즈 자동화 시스템

1. 제품 철학 및 핵심 목표 (Core Objective)
본 시스템은 영업사원이 타겟 상권을 분석하고, 소상공인의 '위기(매출 하락, 업력 장기화에 따른 정체)'와 '기회(풍부한 배후 수요, 높은 면적당 매출액)'를 교차 분석하여, "지금 당장 생활정보홍보우편이 필요한 이유"를 맞춤형 상담 멘트와 제안서로 즉시 출력하는 60:20:20 3단 레이아웃의 B2B 세일즈 웨폰(Weapon)이다.

2. 기존 기능 보존 정책 (Strict Preservation)
행정동 검색 로직: 현재 구현된 지역명(행정동) 텍스트 검색 및 드롭다운 기능은 절대 수정하지 않고 유지한다.

업종 3단 분류 필터: 대분류/중분류/소분류로 이어지는 기존 업종 드롭박스 UI 및 상태(State) 구조는 100% 보존한다.

3. 공공데이터 API 매칭 전략 (Data Pipeline)
우측 패널의 '업종 대분류' 선택 값에 따라 호출하는 API를 동적으로 분기하여 정확한 상권 매출을 가져온다.

외식업 선택 시: /getAreaIndutyAvrOutStats 호출

서비스업 선택 시: /getAreaIndutyAvrSrvcStats 호출

도소매업 선택 시: /getAreaIndutyAvrWhrtStats 호출

타겟 사업자 분석: 리스트에서 특정 업체 클릭 시 /getEvlInfo (개인사업자평가정보) 호출하여 성별, 연령, 설립연도(업력) 데이터 추출.

4. 영역별 상세 구현 명세서 (Component Specifications)
🔴 [Left 영역: 60%] <StrategicMapArea/> (기회와 위기의 시각화)
기존 지도를 유지하되, 평균매출액 API와 배후 수요를 시각적으로 매핑한다.

배후 수요(기회) 폴리곤 렌더링:

선택된 행정동의 '면적단위평균매출금액' 데이터를 기반으로 지도 위에 히트맵(Heatmap) 또는 색상 폴리곤을 오버레이 한다.

UI 효과: 매출 밀집도가 높은 '노른자 상권'은 붉은색으로 렌더링하여 홍보우편 발송 시 ROI가 높음을 직관적으로 보여준다.

타겟 마커 (위기/기회 표시):

단순 마커가 아닌, 업체 상태에 따라 마커 아이콘 색상을 분기한다.

Blue 마커 (기회): 신규 설립 + 고매출 상권 내 위치

Red 마커 (위기): 업력 5년 이상 + 상권 평균 매출 대비 하락세 추정

배후 세대 타겟팅 서클 (Radius):

마커 클릭 시 반경 1km 서클이 그려지며, 중앙에 툴팁으로 "예상 도달 배후 수요: 000 세대 (우편 발송 추천 반경)"을 표시한다.

🟡 [Center 영역: 20%] <SalesPitchGenerator/> (설득의 자동화)
영업사원이 화면을 읽기만 해도 전문적인 컨설팅이 가능하도록 텍스트와 수치를 동적으로 렌더링한다. (특정 사업자 클릭 시 활성화)

섹션 1: 상권 팩트 체크 (API 기반 데이터 패널)

데이터: 지역별 평균매출액 API 3종 활용

UI: 타겟 업체의 지역/업종 '평균 매출금액' 및 '면적단위 평균매출'을 강조된 숫자로 표시.

섹션 2: 맞춤형 상담 멘트 자동 생성 (Pitching Script)

데이터: /getEvlInfo의 '설립명도(업력)', '연령대' + 상권 매출 API 조합

로직 (조건부 텍스트 렌더링):

Case A (기회 - 신규업체 + 고수익상권): "대표님, 현재 [행정동] [업종] 상권은 면적당 매출이 매우 높은 기회의 땅입니다. 오픈 초기 배후 수요 1,000세대를 선점하기 위해 홍보우편 발송이 가장 시급한 타이밍입니다."

Case B (위기 - 노후업체 + 경쟁심화): "대표님, 매장을 운영하신 지 [N]년이 되셨네요. 최근 해당 상권의 [업종] 경쟁이 치열합니다. 기존 이탈 고객을 잡고 동네 상권 점유율을 방어하기 위해 타겟팅 우편 프로모션을 제안합니다."

섹션 3: Action & Proposal (원클릭 제안서)

예상 비용 대비 타겟 상권 객단가를 바탕으로 한 [예상 ROI 시뮬레이터] 표시.

하단에 [맞춤형 영업 제안서 PDF 다운로드] 버튼 배치 (위 데이터와 스크립트가 포함된 화면 캡처 또는 PDF 생성 기능 연동).

🟢 [Right 영역: 20%] <TargetFilterList/> (타겟 추출 및 필터링)
기존의 3단 업종 분류와 행정동 검색을 보존하며, 리스트 UI를 '영업 우선순위' 기반으로 강화한다.

기존 기능 유지: 행정동 검색창, 대/중/소 업종 드롭박스.

리스트 정렬 및 배지(Badge) 시스템 적용:

단순 목록이 아닌, 영업 성공 확률이 높은 순으로 리스트를 정렬.

각 리스트 카드 우측 상단에 /getEvlInfo 데이터 기반의 상태 태그 렌더링.

5. Cursor AI 구현 지시어 (Prompt Instructions for Cursor)
@Cursor:

현재 구현된 행정동 검색 컴포넌트와 업종 3단 분류(대/중/소) 상태(State) 로직은 절대 수정하지 말고 Right 영역(20%) 최상단에 유지할 것.

업종 대분류의 값('외식', '서비스', '도소매')에 따라 Center 영역(20%)에서 호출하는 평균매출액 API 엔드포인트가 동적으로 바뀌도록 useEffect Hook을 구성할 것.

Right 영역의 업체 리스트에서 특정 항목을 onClick 했을 때, 해당 업체의 getEvlInfo 데이터를 Fetching하여 Center 영역의 <SalesPitchGenerator/> 컴포넌트에 Props로 전달하고, 업력(설립명도)에 따라 상담 멘트 텍스트가 조건부 렌더링되도록 구현할 것.

전체 레이아웃은 Tailwind CSS를 사용하여 flex-row, w-[60%], w-[20%], w-[20%] 비율이 정확히 유지되도록 page.tsx의 골격을 짤 것.
