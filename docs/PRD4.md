1. 프로젝트 목표 (Objective)
기존 상권 분석(소진공 API)에 '국토교통부 공동주택(아파트) 단지 및 통계 데이터'를 결합하여, 두루뭉술했던 '예상 배후 수요'를 '실존하는 아파트 단지명과 정확한 세대수'로 전환한다. 이를 통해 홍보우편 발송 수량과 요금을 1원 단위까지 정확하게 산출하는 초정밀 B2B 컨설팅 시스템을 완성한다.

2. API 연동 전략 (Data Pipeline Integration)
① 국토교통부_공동주택 단지 목록제공 서비스 (getLegaldongAptList3 중심 활용)
용도: 특정 상권(법정동/행정동) 내에 존재하는 아파트 단지 목록과 세대수를 추출.

적용 로직: 우측 패널에서 타겟 상가(예: 수성구 수성동4가 치킨집)를 클릭하면, 해당 상가가 속한 법정동 코드를 기반으로 /getLegaldongAptList3 API를 호출하여 반경 1km 내의 아파트 단지 리스트(단지명, 세대수)를 가져온다.

② 국토교통부_공동저장소_통계리스트 (getMolitStatList)
용도: 거시적인 부동산 트렌드(예: 이사/전월세 거래량, 신축 입주 통계 등)를 조회하여 특정 업종의 영업 타이밍 포착.

적용 로직: 가구점, 인테리어, 청소업체, 통신사 대리점 등의 업종을 선택했을 때, 해당 지역의 최근 '부동산 거래/이사 통계'가 상승세라면 "최근 이사/입주 수요 급증 지역 (홍보 최적기)"라는 특별 영업 스크립트를 생성하는 트리거로 사용.

5. 단계별 개발 마일스톤 (Action Plan for Cursor AI)
Step 1 (API 라우팅 구현): app/api/molit/getAptList/route.ts 서버 액션(Route Handler)을 생성하여 국토부 API Key를 숨기고 법정동 코드로 아파트 목록을 JSON으로 파싱.

Step 2 (UI 결합): InsightCenterPanel 내부에 특정 업체가 선택되면 자동으로 해당 법정동의 아파트 리스트를 불러와 세대수를 합산(reduce 함수 사용)하는 로직 구현.

Step 3 (요금 계산기 로직): lib/postal-calc.ts를 만들어 수량에 따른 할인율을 자동 적용하는 함수를 만들고, 이를 제안서 UI와 연동.

Step 4 (지도 시각화): 카카오/네이버 지도 API의 Marker 또는 CustomOverlay를 이용해 아파트 단지들을 맵 위에 시각적으로 띄움.
