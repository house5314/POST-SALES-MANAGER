# 생활정보홍보우편 스마트 영업 네비게이터(PoC) — 심사·실증 요약(개정)

본 문서는 `docs/final.md`의 흐름을 유지하되, **제안서(인쇄/PDF) 업종별 템플릿 분기**를 심사·설문에서 설명할 수 있도록 **변경 요지**를 반영한 개정판입니다. 배포 사이트 중심 서술은 `final.md`와 같습니다.

---

## 1. 제목·개요(요지 동일)

**공공데이터·지도·AI를 활용한 생활정보홍보우편 B2B 스마트 영업 네비게이터 실증(PoC)**  
배포형 웹에서 상권·타겟·견적·제안(PDF)까지 한 흐름으로 시연하며, 가공·실데이터 구분 고지와 선택적 AI·비활성 안내를 갖춥니다.

---

## 2. 변경 요지: 업종별 맞춤 제안서 템플릿(창의성·기술 구현 어필)

### 2.1 판별 순서(`resolveTemplateType`)

제안서 본문 템플릿(A·B·C)을 고를 때 **아래 순서**로 판별합니다.

1. **화면과 동일한 업종 대분류명**(`industryLargeLabel`)과 상가 행의 **`indsLclsCd` 선두 1자**로 먼저 분류합니다.  
2. 그다음 **소분류에 해당하는 `category` 문자열** 키워드(외식·도소매 등)로 보조합니다.  
3. 마지막에 기존과 같이 **매출 추세(`revenueTrend`)·업력** 휴리스틱을 적용합니다.

이 순서로 **“공공 상가 API가 주는 업종 코드·화면 필터 선택”**과 **“제안서에 쓰는 스토리 템플릿”**이 맞물린다고 설명할 수 있습니다.

### 2.2 분기 규칙(요약)

| 템플릿 | 성격 | 판별 요지 |
|--------|------|-----------|
| **A** | 외식·기회 창출형 | 대분류명에 **외식** 또는 **음식**이 포함되거나, 대분류 코드 선두가 **`I`**(숙박·음식 등 상가(상권)정보 체계에 따른 값). |
| **B** | 도소매·위기 극복형 | 대분류명에 **도소매** 또는 **소매**가 포함되거나, 코드 선두가 **`G`**. |
| **C** | 서비스·생활밀착형 | 대분류명에 **서비스**가 포함되거나, 코드 선두가 **`L`, `M`, `N`, `P`, `Q`, `R`, `S`** 중 하나. 코드 집합은 **공공데이터포털의 상가(상권)정보 업종코드 자료**와 대조·조정할 수 있음을 주석으로 안내합니다. |

**서비스(C)** 분기는 **매출·업력 분기보다 앞**에 둡니다. 그래서 서비스업인데 업력만 길어 **도소매형(B)**으로 잘못 가던 경우를 줄입니다.

### 2.3 화면과 제안서의 일치(`industryLargeLabel`)

- 제안서 출력 시 옵션 **`ProposalPrintExtras.industryLargeLabel`**에 **화면 필터에서 고른 업종 대분류명**(`selectedLargeName`)을 넘깁니다.  
- **`InsightCenterPanel`**에서 제안서 생성 버튼 호출 시 위 값을 넣어, **필터에서 선택한 대분류**와 **PDF 템플릿 분기**가 같아지도록 했습니다.  
- **`buildTemplateContent`**는 위 라벨을 받아 **`resolveTemplateType`**에 전달합니다.

### 2.4 유지보수 메모

- 실제 **`largeUpjongList`**(또는 동등 API) 응답이 기관·개정으로 바뀔 수 있으므로, 서비스업 코드 집합(`SBIZ_SERVICE_INDS_LCLS_FIRST` 등)은 **포털 업종코드 자료**에 맞춰 조정하면 됩니다.  
- **`npm run lint`** 기준으로 스타일·훅 규칙은 통과한 상태를 전제로 합니다.

## 3. `final.md`와의 관계

- **현황·한계·운영·기대효과·시연 체크리스트** 등 나머지 서술은 **`docs/final.md`**를 그대로 인용해도 됩니다.  
- 본 **`final2.md`**는 그중 **“업종별 맞춤 제안서 템플릿”**을 심사·설문 게시판에 짧게 붙일 때 쓰는 **개정 보조** 용도입니다.

## 부록: 심사 시 업종 템플릿 질문 예시

1. 필터에서 **외식 / 도소매 / 서비스** 대분류를 바꾼 뒤 제안서를 열었을 때, **제목·진단 문구 톤**이 업종에 맞게 바뀌는가?  
2. 화면에서 고른 대분류와 PDF 상단·본문의 **템플릿 라벨(예: Template A·B·C)** 이 **일치**하는가?  
3. 서비스 업종을 고른 경우, **업력이 길다고 해서** 도소매형 위기 극복 톤으로 **끌려가지 않는가?**
