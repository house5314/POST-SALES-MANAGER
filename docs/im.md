# 우정사업본부 AI 과제 개발 내용 정리

본 문서는 우정사업본부의 **「2026 AI 혁신 아이디어 공모전」** 및 **「AI 업무 툴 직접 개발」** 부문에 제출하기 위한 과제 개발 내용을 체계적으로 정리한 문서입니다. 본 문서를 바탕으로 Cursor AI 등 개발 툴을 활용하여 실제 구현 및 문서화를 진행할 수 있습니다.

## 1. 공모전 심사 기준 및 배점 (목표 지향적 개발)

본 과제는 심사 기준표의 각 항목에서 고득점을 획득하기 위해 전략적으로 기획 및 개발되었습니다.

### 1차 심사: AI 서류심사 (100점 만점)
* **창의성 및 독창성 (20점):** 기존 우체국 서비스(단순 우편 발송) 대비 참신성, 새로운 발상 여부. -> *단순 정보 제공을 넘어, 상권 분석 데이터(공공 API)와 결합한 맞춤형 B2B 마케팅 컨설팅 툴 제안으로 독창성 확보.*
* **기술적 구현력 (30점):** 문제 정의 -> 해결 방안 -> 기대 효과 등 알고리즘 설계의 논리적 타당성 및 데이터의 정확성. -> *소진공 API, 국토부 API 연동 및 자동 견적 산출 로직 구현으로 높은 기술력 입증.*
* **AI 활용 최적화 (30점):** 업무 수행 과정에서의 AI 기반 서비스 활용 여부, 개발 단계 전반에 AI 도구 적극 적용 여부. -> *Cursor AI를 활용한 신속한 개발 프로세스(Human-AI 페어 프로그래밍) 및 자동화된 제안 멘트 생성 기능 강조.*
* **기대효과 및 확장성 (20점):** 실현 가능성, 적용 대상 확장 가능성, 지속 운영 가능성. -> *전국 모든 상권/우체국에 즉시 적용 가능하며, CRM 기능 연계 시 지속 운영 가능한 모델.*

### 2차 심사: 종합 평가 (100점 만점) + 가점 (10점)
* **직원 설문 (30점):** 현업 적용 가능성(효용성 15점), 사용 기대감(이용 의향 15점). -> *영업 현장의 실질적 Pain point 해결(상권 분석, 견적 산출 자동화).*
* **내·외부 위원 평가 (40점):** 목표 부합성(10점), 확산성·효과성(10점), 실현가능성(10점), 창의성(10점).
* **AI 공개 심사 (30점):** 완성도(10점), 기술성·혁신성(10점), 안전성·윤리성(10점).
* **가점 요소 (최대 10점):** **내부망 환경에서의 즉시 구동 가능성**, **실증·PoC 가능한 프로그램**. -> *실제 구동 가능한 웹 애플리케이션(PoC) 완성으로 가점 확보.*

## 2. 과제 개발 내용 (신청서 작성용 초안)

다음은 공모신청서(참고 4)의 '과제 개발 내용' 양식에 맞춘 초안입니다.

### ① 제 목
[우편 매출 혁신] 공공데이터 상권 분석 기반 '생활정보홍보우편 스마트 영업 네비게이터' 구축 및 실증(PoC)

### ② 개 요
우체국 '생활정보홍보우편' B2B 영업 시, 영업 직원의 경험과 감에 의존하던 방식을 탈피하여, 공공데이터(소상공인시장진흥공단, 국토교통부) 기반의 객관적 상권 분석과 정밀한 타겟팅(배후 세대수)을 제공하는 웹 기반 맞춤형 컨설팅 시스템입니다. 이를 통해 현장 영업 직원은 타겟 업체 선정부터 맞춤형 스크립트, 예상 견적, ROI 시뮬레이션이 포함된 제안서를 즉시 생성하여 계약 성공률을 획기적으로 높일 수 있습니다.

### ③ 개발기간
약 3주 (2026. 5. 1. ~ 2026. 5. 22.)

### ④ 현재상태
☑ 구현완료  □ 로컬운영  □ 내·외부 서버 배포하여 운영 중

### ⑤ 개발 목적
* **업무 문제점:** 기존 생활정보홍보우편 영업은 대상 업체 발굴과 타겟팅(배달 세대수 산정), 효과성 입증에 많은 시간과 노력이 소요되었으며, 영업 직원의 개인 역량에 크게 의존했습니다.
* **해결 목적:** 데이터 기반의 객관적인 상권 분석 자료와 비용 대비 효과(ROI)를 자동 산출하여 제공함으로써, 영업 직원의 업무 부담을 줄이고 제안의 신뢰도를 높여 우편 매출 증대에 기여하고자 합니다.

### ⑥ 사용 AI 프로그램·도구
* **Cursor AI (Claude 3.5 Sonnet 등 LLM 기반):** 프론트엔드 UI 컴포넌트 생성, 공공데이터 API 연동(Fetch) 로직 작성, 요금 계산 알고리즘 및 상태 관리(React Hooks) 구현, 제안서 PDF 출력 기능 개발 등 전반적인 코딩 및 리팩토링 과정에서 페어 프로그래머로 활용.

### ⑦ 개발 방법
1.  **업무 문제점 분석:** 현장 영업 시 발생하는 데이터 확보의 어려움 및 제안서 작성 시간 과다 문제 도출.
2.  **데이터 소스 연계 기획:** 소상공인시장진흥공단(상가정보, 평균매출 등) 및 국토교통부(아파트 단지 목록, 세대수) 공공 API 활용 방안 설계.
3.  **UI/UX 및 기능 설계:** 지도 기반 상권 시각화(Left 60%), 동적 제안서 생성 및 요금 계산(Center 20%), 필터링 및 리스트(Right 20%)의 3단 레이아웃 기획.
4.  **AI(Cursor) 페어 프로그래밍:** 설계안(PRD)을 Cursor AI에 프롬프트로 입력하여 Next.js, React 기반의 웹 애플리케이션 코드를 신속하게 자동 생성 및 수정.
5.  **테스트 및 보완:** 로컬 환경에서 API 연동 테스트 및 실시간 요금 계산, PDF 렌더링 기능 검증 후 고도화.

### ⑧ 개발 내용·단계별 로드맵
* **1단계 (설계 및 API 연동):** UI 레이아웃 구성, 네이버 지도 API 연동, 소상공인 공공데이터 API 호출 모듈 개발.
* **2단계 (기능 고도화):** 타겟 아파트 세대수 기반 수량 산출, 우편 요금 및 감액률(30%) 자동 계산 로직 적용.
* **3단계 (제안서 자동화):** 업력/업종별 맞춤형 영업 스크립트 동적 생성, 인쇄(PDF)용 제안서 레이아웃 최적화.
* **4단계 (통합 및 PoC):** '소상공인365' 전문 상세 분석 iframe 위젯 통합, 전체 시스템 구동 테스트 완료.

### ⑨ 기능 및 내용
1.  **상권 시각화 및 타겟 추출:** 지도 상에 타겟 상권 및 아파트(배후수요)를 표시하고 실시간 세대수 합산.
2.  **맞춤형 영업 스크립트 생성:** 대상 업체의 업력, 매출 변동 추이 등에 맞춰 '위기 극복형', '기회 창출형' 등 설득력 있는 상담 멘트 제공.
3.  **실시간 요금 견적기:** 타겟 수량 및 우편물 형태(접착/봉투/봉입)에 따른 기준 요금 및 특별 감액(30%)이 적용된 최종 견적가 자동 산출.
4.  **맞춤형 제안서(PDF) 즉시 출력:** 지도, 상권 분석 지표, 스크립트, 견적, 소상공인365 상세 차트(부록)가 포함된 전문가 수준의 브리핑 자료 원클릭 생성.

### ⑩ 설계서
(별도 첨부된 아키텍처 및 PRD 파일 참조 - 프론트엔드 구조, API 연동 다이어그램 등)

### ⑪ 활용 효과 (기대효과)
* **소요시간 (개선 전 -> 개선 후):** 상권 분석 및 맞춤형 제안서 작성 (약 2시간 -> **1분 이내**)
* **오류율 (개선 전 -> 개선 후):** 수기 요금 계산 및 타겟 세대수 오차 발생 (빈번 -> **0%**, 시스템 자동 계산)
* **만족도:** (데이터 기반의 전문적인 제안서 제공으로) 고객(소상공인)의 신뢰도 상승 및 영업 직원의 업무 피로도 대폭 감소.
* **정성적 효과:** '감'에 의존하던 영업 방식에서 탈피하여 '데이터 기반(Data-Driven)' 마케팅으로 우체국 B2B 영업의 패러다임 전환, 우편물 접수 물량 증대에 따른 수익성 강화.

### ⑫ 자동 검증(PoC 환경 변수)
* **목적:** 공모·실증 전 데모 실패(키 누락·공공 serviceKey 혼동)를 줄이고, 심사·내부 문서에서 “자동 검증”을 **실행 명령으로 입증**합니다.
* **명령:** `npm run validate:env` — `.env.local`의 **필수** `PUBLIC_DATA_API_KEY`, `NEXT_PUBLIC_NAVER_CLIENT_ID` 점검 및 공공 API 샘플 호출(기본). `npm run verify:public-data`는 동일 스크립트 호출을 유지하는 **호환 별칭**입니다.
* **옵션:** `node scripts/validate-env.mjs --no-network` — 네트워크 검증 없이 키 존재만 확인(오프라인·CI 스모크).
* **빌드 연동:** `npm run build` 직전에 `prebuild`로 `validate:env`가 실행됩니다. **CI·Vercel** 등에서 `.env.local` 파일이 없으면 검증을 건너뜁니다(`CI` / `VERCEL` / `SKIP_ENV_VALIDATION`). 로컬 PoC는 반드시 `.env.local`을 두세요.
* **구현 위치:** `scripts/validate-env.mjs`(본체), `scripts/verify-public-data-keys.mjs`(위임 진입점), GitHub Actions `.github/workflows/verify-env.yml`(스크립트 스모크).

### ⑬ AI 입력 마스킹·로그 정책(안전성·윤리 대응)
* **목적:** 외부 LLM으로 사용자 입력을 보낼 때 **원문 로그 금지·민감 패턴 사전 제거**를 코드로 고정하여, 2차 심사의 **안전성·윤리** 및 내부 감사 요구에 대응합니다.
* **마스킹:** `lib/ai/mask-sensitive-text.ts` — `maskSensitiveTextForModel()` 로 이메일·한국 전화·주민번호형·긴 숫자열 등을 토큰 치환 후에만 모델 `prompt`에 전달합니다.
* **로그:** `lib/ai/ai-request-logger.ts` — `logAiRouteEvent()` 가 프로덕션에서는 **메타데이터(JSON)** 만 출력하고, 개발 환경에서만 마스킹된 문자열의 **짧은 미리보기**를 허용합니다.
* **API:** `POST /api/ai/assist` (`app/api/ai/assist/route.ts`) — 요청 본문의 `prompt`에 위 정책을 적용한 뒤, `GOOGLE_GENERATIVE_AI_API_KEY`가 있을 때만 `generateText` 호출; 키 미설정 시 **503** 및 한글 안내(원문 비노출).

### ⑭ 내부망·모델 API 경로(가점·운영 대응)
* **목적:** 폐쇄망·DMZ에서 **어떤 URL로 모델을 호출하는지** 환경 변수와 코드로 명시하여, “내부망 모델 경로 문서 미확인” 지적에 대응합니다.
* **환경 변수:** `GOOGLE_GENERATIVE_AI_BASE_URL`(선택·프록시 접두), `GOOGLE_GENERATIVE_AI_MODEL`(선택), `GOOGLE_GENERATIVE_AI_API_KEY`(서버).
* **코드:** `lib/ai/google-genai-config.ts` — `createGoogleGenerativeAI({ apiKey, baseURL })` 에 `baseURL`을 넘기는 분기.
* **상세 문서:** [`docs/ai-internal-network.md`](./ai-internal-network.md) — 공인망 기본 접두, 내부 프록시, Google 미사용 시 로드맵을 표로 정리.

### ⑮ 운영 주체·데이터 갱신·API 쿼터(지속 운영·심사 대응)
* **목적:** “운영 주체·데이터 갱신 주기·API 쿼터 명시 미확인” 지적에 대응하여, PoC와 상용 전환 시의 **역할·데이터 신선도·호출 한도 근거**를 문서·코드로 고정합니다.
* **운영 주체:** PoC는 과제 실증용이며, 상용 시 **도입 우정청(또는 본부 지정 부서)** 가 소유·배포·민원을 담당한다는 전제를 명시합니다(`lib/operations/governance-copy.ts` 의 `OPERATING_ENTITY_POC_LINE`).
* **데이터 갱신:** (1) 소진공·국토부 등 **원천 DB 갱신**은 각 기관 정책, (2) 화면 수치는 **API 조회 시점 스냅샷**(PoC는 사용자 조회 위주, 서버 캐시 비필수)임을 구분합니다.
* **API 쿼터:** `apis.data.go.kr`·NCP·Google 등은 **발급 계약·포털 안내**가 우선이며, 상가 목록 등은 **필터·동 변경 시**에만 재호출되도록 설계되어 있습니다(과도 폴링 없음).
* **가공 매출·상권 지표 UI:** `MARKET_STAT_METRICS_SOURCE_LABEL` 등은 **제안서 생성(인쇄/PDF) 카드 하단**·「매출 추세」카드·필터·제안서 PDF 상단에 노출되어 시연 시 심사관이 확인할 수 있습니다(`lib/commercial-api/mock-sales-metrics.ts` 상수·`InsightCenterPanel`).
* **상세 문서:** [`docs/operations-data-quota.md`](./operations-data-quota.md) — 표·체크리스트·외부 참조 링크 안내.
